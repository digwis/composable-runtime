# 进化运行时 v2 — 全面超越 ARISE/Evolver/Hermes

## 设计目标

借鉴 ARISE 的失败驱动进化、Evolver 的审计性、Hermes 的多层记忆，
结合我们已有的基因组/谱系/交叉重组生物学隐喻，构建一个系统级自进化运行时。

## 核心差异（vs 竞品）

| 特性 | ARISE | Evolver | Hermes | 我们 v2 |
|---|---|---|---|---|
| 进化对象 | 可执行工具 | Prompt | Prompt+Skill | **可执行能力+基因组** |
| 触发方式 | 失败驱动 | 手动/定时 | DSPy优化 | **失败驱动+定时+手动** |
| 沙箱验证 | Docker/subprocess | 无 | 无 | **subprocess+资源限制** |
| A/B测试 | ✅ | ❌ | ❌ | **✅ 适应度对比** |
| 交叉重组 | ❌ | ❌ | ❌ | **✅ 基因组交叉** |
| 谱系追踪 | 版本控制 | EvolutionEvent | 无 | **lineage+generation** |
| 多层记忆 | ❌ | memory/目录 | 三层 | **三层（短期/长期/进化）** |
| 系统级常驻 | ❌ | ❌ | ✅ | **✅ daemon+PATH注入** |
| 多LLM后端 | OpenAI | OpenAI | Nous | **HTTP+CLI(devin/claude)** |
| MCP协议 | ❌ | ❌ | ❌ | **✅** |
| 语言 | Python | JS | Python | **Rust** |

## 新增模块

### 1. `daemon.rs` — 系统级常驻服务
- Unix socket server (本地 IPC)
- MCP server (stdio, AI 工具接入)
- PATH 注入 (~/.orch/bin/ 能力可执行文件)
- 后台进化循环 (定时检测缺口+变异+淘汰)
- LLM 后端自动发现
- 信号处理 (graceful shutdown)

### 2. `sandbox.rs` — 沙箱验证
- subprocess 隔离执行
- CPU/内存/时间资源限制
- 输出捕获 + 结构化验证
- 对抗测试 (边界输入、异常输入)
- 安全检查 (禁止危险操作)

### 3. `ab_test.rs` — A/B 测试
- 新旧能力并行执行相同输入
- 适应度对比 (成功率/延迟/输出质量)
- 统计显著性检验
- 自动提升/回滚

### 4. `failure_driver.rs` — 失败驱动进化
- 拦截任务执行失败
- 分析失败原因 (LLM 归因)
- 检测能力缺口
- LLM 合成新能力
- 沙箱验证 → A/B 测试 → 注册

### 5. `memory.rs` 改进 — 多层记忆
- 短期: 当前任务上下文 (HashMap, in-process)
- 长期: 成功/失败模式 (SQLite)
- 进化: 基因组+谱系 (genomes.json, 已有)

## 实现顺序

1. daemon.rs — 基础常驻服务
2. sandbox.rs — 沙箱验证
3. failure_driver.rs — 失败驱动进化
4. ab_test.rs — A/B 测试
5. memory.rs 改进 — 多层记忆
6. main.rs — daemon 子命令
7. 集成测试
